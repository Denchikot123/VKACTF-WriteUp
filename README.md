### Вход на сервер

Для подключения к серверу был скачен оригинальный Quake 3 Arena с GitHub для удобства администрирования.

Перед управлением сервером и подключением был изучен набор команд из сайта http://www.joz3d.net/html/q3console.html

В нем были найдены такие команды как:

- /connect [IP] - Подключение к серверу по IP адресу
- /rcon [CMD] - Удаленное управление сервером из консоли в игре
- /rconPassword - Вход в удаленное управление по паролю
- /rcon cmdlist - Список команд для удаленного управления

После подключения к серверу через терминал игры ( ~ or ` ) мы видим перед собой карту категории Capture The Flag - Space_CTF

Изначально доступ без пароля в Remote Control заблокирован. Изучим найденный в начале репо с дефолтными файлами, конфигурациями сервера.

<img width="458" height="121" alt="изображение" src="https://github.com/user-attachments/assets/c0444630-2a94-4b7a-a777-0492490302ad" />

Увидим что файл server.cfg содержит в себе параметр **(seta rconpassword "secret")**, что является паролем для Remote Control - **secret**

Данный пароль подошел к нашему серверу и мы получили доступ к команде /rcon

### Path Traversal

Изучив команды из /rcon cmdlist были использованы команды для вывода информации: serverinfo, status, cvarlist, exec:

- serverinfo - Выдает информацию о сервере (Движок, время жизни и т.д.)
- status - Выводит информацию о игроках на сервере (ID, Name, Score и т.д.)
- cvarlist - Вывод всех консольных переменных
- exec - Вывод содержимого файла (текст)

Все команды сработали штатно и вывели информацию о конфигурациях и его содержимом, что подтвердило уязвимость - **Path Traversal**. Но файлов было много и флаг мог находится где угодно

Поэтому изначально была использована команда - /rcon setenv **[ИМЯ_ПЕРЕМЕННОЙ] [ЗНАЧЕНИЕ]**

Команда направлена на установку или изменение переменных окружения операционной системы, на которой физически запущен выделенный сервер игры. При указании лишь имени переменной, выводится его значение

Предположим что наш флаг имеет имя переменной FLAG
